<h2>ServiceDesk</h2>

<p>Приложение для работы с пользовательскими обращениями, 
которое фиксирует их в базе данных и предоставляет возможность 
обработки сотрудниками поддержки.</p>

<h3>Пользовательский флоу</h3>

<ol>
<li>Пользователь пишет сообщение в Telegram</li>
<li>Пользователю приходит автоответ, что сообщение в работе</li>
<li>Пользователь получает сообщения от оператора</li>
<li>После закрытия обращения, пользователю приходит сообщение, что проблема решена</li>
</ol>

<h3>Флоу оператора</h3>

<ol>
<li>На главной странице есть список обращений с возможностью фильтрации по статусу и сортировки по времени создания</li>
<li>Оператор может назначить обращение на себя и ответить пользователю через интерфейс в Telegram</li>
<li>По завершению работы оператор закрывает обращение</li>
<li>Новые сообщения от пользователя поступают в новое обращение</li>
</ol>

<h3>Стек технологий</h3>

<ul>
<li><strong>Backend:</strong> FastAPI</li>
<li><strong>Telegram API:</strong> Aiogram 3 + технология вебхуков</li>
<li><strong>ORM:</strong> SQLAlchemy</li>
<li><strong>База данных:</strong> SQLite</li>
<li><strong>Миграции:</strong> Alembic</li>
</ul>
